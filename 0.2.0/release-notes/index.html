    Release notes

    The Operator-Framework community is excited to announce the v0.2.0 release of operator-controller!

    With this release of operator-controller, you can install an operator from a catalog of operators.

    Example workflow of installing an operator from a catalog of operators

    Create a Kind cluster:

    $ kind create cluster 
Creating cluster "kind" ...

 ✓ Preparing nodes 📦  
 ✓ Writing configuration 📜 
 ✓ Starting control-plane 🕹️ 
 ✓ Installing CNI 🔌 
 ✓ Installing StorageClass 💾 
Set kubectl context to "kind-kind"
You can now use your cluster with:

kubectl cluster-info --context kind-kind

    Add a catalog of operators to the cluster to make the operators shipped in the catalog available for installation on the cluster. This is similar to how a repository needs to be added to the list of repositories available for a package manager to install from, when you want to install a binary available in the repository with the help of the package manager. We'll use the upstream community-operators catalog quay.io/operatorhubio/catalog: 

    $ kubectl apply -f - <<EOF
apiVersion: catalogd.operatorframework.io/v1alpha1
kind: Catalog
metadata:
  name: operatorhubio
spec:
  source:
    type: image
    image:
      ref: quay.io/operatorhubio/catalog:latest
EOF

    Installing specific versions of an operator

    By default, the latest version of the operator is installed when a version is not specified in the Operator CR. We can confirm this by matching the image reference from the installedBundleResource field with the image reference for the latest version, which in argocd-operator's case, is v0.6.0: 

    $  kubectl get bundlemetadata operatorhubio-argocd-operator.v0.6.0 -o yaml | grep image 
  image: quay.io/operatorhubio/argocd-operator@sha256:1a9b3c8072f2d7f4d6528fa32905634d97b7b4c239ef9887e3fb821ff033fef6
  - image: gcr.io/kubebuilder/kube-rbac-proxy:v0.8.0
  - image: quay.io/argoprojlabs/argocd-operator@sha256:99aeec24cc406d06d18822347d9ac3ed053a702d8419191e4e681075fed7b9bb
  - image: quay.io/operatorhubio/argocd-operator@sha256:1a9b3c8072f2d7f4d6528fa32905634d97b7b4c239ef9887e3fb821ff033fef6

    Looks like the last image reference is in fact the one that is also visible in the installedBundleResource field.

    We can also pin the installation of an operator to a specific version, if we wish to install something other than the latest: 

    $ kubectl apply -f - <<EOF
apiVersion: operators.operatorframework.io/v1alpha1
kind: Operator
metadata:
  name: argocd-operator
spec:
  packageName: argocd-operator
  version: v0.5.0
EOF
operator.operators.operatorframework.io/argocd-operator created

$ kubectl get operator argocd-operator -o yaml | grep status -A 4
